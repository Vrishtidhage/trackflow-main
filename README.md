# TrackFlow

TrackFlow is a bug tracking application built with React, Vite, FastAPI, and SQLAlchemy.

## Project Structure

```text
TrackFlow-main/
|-- client/
|   |-- index.html
|   |-- package.json
|   |-- vite.config.js
|   |-- .env.example
|   `-- src/
|       |-- api.js
|       |-- App.jsx
|       |-- main.jsx
|       |-- components/
|       |-- contexts/
|       |-- pages/
|       |-- theme/
|       `-- utils/
|-- server/
|   |-- requirements.txt
|   |-- .env.example
|   |-- app/
|   |   |-- main.py
|   |   |-- config.py
|   |   |-- database.py
|   |   |-- models/
|   |   |-- routers/
|   |   |-- schemas/
|   |   |-- services/
|   |   `-- utils/
|   `-- alembic/
`-- render.yaml
```

`client/dist/` is generated by `npm run build`. Do not edit files inside `dist`.

## Local Run Commands

Install dependencies once:

```powershell
cd client
npm install
cd ..\server
.\venv\Scripts\pip.exe install -r requirements.txt
```

Run backend:

```powershell
cd server
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Run frontend:

```powershell
cd client
npm run dev -- --host 127.0.0.1 --port 5173
```

Open:

```text
Frontend: http://127.0.0.1:5173
Backend:  http://127.0.0.1:8000
```

## Render Deployment

This repo includes `render.yaml` for Render Blueprint deployment. The Blueprint creates:

```text
trackflow-db      Render PostgreSQL database
trackflow-api     FastAPI backend web service
trackflow-client  React frontend Node web service
```

Deploy steps:

```text
Render Dashboard -> New -> Blueprint -> connect this GitHub repo -> Apply
```

The Blueprint wires these automatically:

```text
DATABASE_URL=<trackflow-db internal connection string>
SECRET_KEY=<generated by Render>
VITE_API_URL=<trackflow-api external URL>
```

Set this backend secret in Render when prompted:

```text
ADMIN_PASSWORD=<strong admin password>
```

Default admin email:

```text
ADMIN_EMAIL=admin@trackflow.local
```

Backend service settings from `render.yaml`:

```text
Root Directory: server
Build Command: pip install -r requirements.txt
Start Command: PYTHONPATH=/opt/render/project/src/server uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

Frontend service settings from `render.yaml`:

```text
Root Directory: client
Build Command: npm ci && npm run build
Start Command: npm start
```

Do not use `http://127.0.0.1:8000` in production. That only works locally.
